# 模型库 — 优化类（CO-STAR格式 · DeepSeek V4 Pro）

## 适用场景
- 资源分配、路径规划、排班调度、投资组合
- 核心特征：明确的**目标函数**和**约束条件**，找最优解

## 方法速查

| 方法 | 适用 | 何时选用 |
|------|------|---------|
| **LP/MILP** | 线性问题 | 精确解可用，优先使用 |
| **DP** | 多阶段 | 最优子结构 |
| **GA/PSO/SA** | NP-hard | 精确不可解时 |

## 完整Prompt模板（复制使用）

```
Context:
你正在解决一个数学建模竞赛的优化问题。题目提供了完整的背景、数据和约束条件。
你是全球顶尖的运筹学专家，专门为竞赛设计求解方案。
本问题类型：{LP / MILP / NLP / 组合优化}

Objective:
为以下问题建立完整的优化数学模型，并选择合适的求解方法。

Constraints (约束):
- 决策变量必须明确定义类型（连续/0-1/整数）
- 目标函数必须用标准数学符号写出
- 所有约束条件逐条编号，用公式表示
- 参数来源必须标注（题目给定/数据估计/文献参考）
- 如果问题NP-hard，必须给出启发式方法的理由

Output Format:
1. 【变量定义】
   表格：符号 | 含义 | 类型 | 取值范围
   
2. 【目标函数】
   (完整公式，标注最大化/最小化)
   
3. 【约束条件】
   约束1：(公式) — 含义
   约束2：(公式) — 含义
   ...
   
4. 【问题类型判断】
   属于XX问题，因为YY。复杂度估计：O(...)
   
5. 【求解方法选择】
   方法A — 理由 — 复杂度
   方法B — 理由 — 复杂度
   推荐方法：X — 原因
```

## 常见坑
- 混淆最大化/最小化导致目标函数符号反
- 遗漏隐式约束（如非负约束）
- NP-hard问题强求精确解 → 实际不可行
- 遗传算法的编码方式与问题不匹配

## 论文写作要点
- 三要素必完整：目标函数+决策变量+约束条件
- 约束条件编号便于正文引用
- 算法参数表：参数名、含义、取值、选取依据
