# 模型库 — 评价类（CO-STAR格式 · DeepSeek V4 Pro）

## 适用场景
- 方案评价、指标权重、综合排名、绩效评估
- 核心特征：多对象+多指标，需要综合打分

## 方法速查

| 方法 | 主/客观 | 何时选用 |
|------|--------|---------|
| **熵权法** | 客观 | 数据充分，避人为偏见 |
| **AHP** | 主观 | 专家判断，方案≤7 |
| **CRITIC** | 客观 | 考虑指标冲突性 |
| **TOPSIS** | 均可 | 需要排名，接近理想解 |

## 组合策略（国一常见）
- **熵权法+TOPSIS**（最经典）：客观权重→贴近度排序
- **AHP+模糊评价**：主观权重→隶属度打分
- **CRITIC+TOPSIS**：冲突性权重（比熵权法更全面）

## 完整Prompt模板（复制使用）

```
Context:
你正在解决一个数学建模竞赛的评价问题。有多个评价对象和多个评价指标（含正向和逆向）。
你是评价建模专家，需要建立完整的综合评价体系。

Objective:
建立评价指标体系→确定权重（至少2种方法比较）→综合评价（TOPSIS）→输出排名+敏感性分析。

Constraints:
- 指标必须区分正向/逆向，做无量纲化
- 权重至少用2种方法（如熵权法+CRITIC），对比结果
- TOPSIS完整推导链：加权矩阵→正负理想解→距离→贴近度
- 必须做敏感性分析：改变某一指标权重±20%，看排名变化

Output Format:
1. 【评价指标体系】
   表格：指标 | 含义 | 正/逆向 | 数据来源
   
2. 【权重确定】
   方法1熵权法：信息熵公式→熵值→权重（完整推导）
   方法2CRITIC：冲突性+波动性→权重
   对比表：指标 | 熵权法权重 | CRITIC权重 | 差异
   
3. 【TOPSIS综合评价】
   加权矩阵→正负理想解→距离→贴近度
   排名表：对象 | 贴近度 | 排名
   
4. 【敏感性分析】（🇸 论文灵魂）
   指标X权重±20% → 排名变化表
   稳定性结论：XX指标对排名影响最大/最小
```

## 常见坑
- 熵权法中信息熵公式写错 → P_ij不能为0（加小常数）
- TOPSIS未区分正向/逆向指标
- 仅用一种方法确定权重 → 被认为不严谨
- 不做敏感性分析 → 评委直接扣分

## 论文写作要点
- 熵权法完整推导不可省略（信息熵→权重）
- TOPSIS完整推导不可省略（矩阵→理想解→贴近度）
- 权重对比表是亮点
- 敏感性分析是国一标志
