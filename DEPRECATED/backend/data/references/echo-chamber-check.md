# 回音室检查报告 — Lemma 2024C/2025B

> 日期：2026-06-20
> 依据：追问协议（UF-PROTO-001）§9 回音室风险
> 要求：每次复盘必答——"这次有'用户接受了，但事后证明是错的'的案例吗？"

## 发现：3 个"用户接受但后来被推翻"的案例

### 案例 1：V8.1 校准系统 — 循环论证（🔴 最严重）

**时间**：2026-06-19

**当时接受了什么**：
- V8.1 校准系统设计了 KNOWN_RANKINGS（15 篇论文的"真实等级"标签），用于测量审稿 Agent 的预测准确率
- 报告准确率：53%（8/15 正确）
- 当时认为这个数字有意义，作为 V8.1 的验证指标

**为什么是错的**：
- `KNOWN_RANKINGS` 的"真实等级"标注来源是**审稿 Agent 自身的判定 + 已知信息交叉验证**
- 这是**循环论证**：用 Agent 判定的等级作为 ground truth，再测量 Agent 的准确率
- 53% 这个数字本质是 **self-report 的变体**，不是真实的外部验证

**怎么发现的**：
- V8.3 校准系统重写时，审查 `calibrate.py` 的 KNOWN_RANKINGS 来源
- 发现标注方法里写着"基于审稿 Agent 判定 + 已知信息交叉验证"
- 认识到这是循环论证后，V8.3 改为只测重测信度 + 区分度

**教训**：
- LLM 生成的标签不能当作 ground truth——这是置信度标定协议（UF-PROTO-004）的核心原理
- 应该在设计校准系统时就意识到这一点，而不是跑了 15 篇论文之后才发现

### 案例 2：7 项交叉验证 — 假"通用"（🟠 中等）

**时间**：2026-06-16 → 2026-06-19

**当时接受了什么**：
- 基于 2025B（频域分析题）设计了一套"7 项强制交叉验证"检查点
- 作为标准流程写入 CLAUDE.md 和 Agent R prompt

**为什么不够好**：
- 2024C（优化题）实测中，7 项中的 5 项（#3-#7）完全不适用
- #3 权重定义偏离、#4 m₀ 范围、#5 Bootstrap CI、#6 谐波检测、#7 m₀ 范围矛盾——全是频域分析的专属检查
- Agent R 在 Phase 1 中不得不手工补充 4 项自定义检查（轮作/豆类/滞销/水浇地）

**怎么发现的**：
- 2024C Phase 1 实测时 Agent R 报告"5/7 项 N/A"
- V9.0 引入题型感知的交叉验证模板（`validation-templates.md`）

**教训**：
- 从一个数据点（2025B）推广到"通用"是危险的
- 应该在设计时就用多种题型测试，而不是信了"7 项强制"的强制两个字

### 案例 3：Phase 1 初裁 — Lead 的判断力被高估（🟡 轻度）

**时间**：2026-06-19

**当时接受了什么**：
- 2024C Phase 1 辩论结束后，Lead 做了初次裁定
- 初裁看起来合理，进入了 Phase 2

**为什么不够好**：
- Agent R 的辩论对比（`AgentR_辩论对比.md`）发现初裁**遗漏了 4 个 P0 致命问题**
- 最关键的遗漏：C4 约束（∑y≤1）直接违反题目"每季可以合种不同的作物"
- 如果没被 R 复查发现，Phase 2 的代码会基于错误的模型——全部白跑

**怎么发现的**：
- R 在 Phase 1 收尾时主动对比了初裁和自己的推导
- 发现了"初裁遗漏的关键修正"，Lead 更新为终裁（F1-F6）

**教训**：
- Lead 的裁定本身也是需要被审查的——不能假设 Lead 不会漏
- 这个案例直接支持了信息不对称协议（UF-PROTO-002）的设计：多一个不看全貌的视角，就能发现看全貌的人漏掉的东西

## 对回音室协议的启示

| 发现 | 对应协议问题 |
|------|------------|
| 案例 1 | 置信度标定 — LLM 的自我报告不能当证据 |
| 案例 2 | 追问协议 — 从单点数据推广时要追问"这真的通用吗" |
| 案例 3 | 收敛失败 — Lead 裁定需要被审查，不是最终答案 |

这三个案例都不是"用户认知盲区"导致的问题（回音室风险的定义），而是**系统设计阶段的过度自信**——接受了第一个看起来合理的方案，没有追问"这个真的对吗"。

**对自适应信任阈值的提示**：如果信任阈值基于"用户接受/否决"的反馈来调整，而用户自己也有过度自信的倾向——那么系统会学着在用户自信时也自信，而用户最自信的时候恰恰是案例 1（53% 准确率看起来不错）和案例 2（7 项检查听起来完善）——这些正是最需要警惕的时刻。
