# 模型库 — 图论类（CO-STAR格式 · DeepSeek V4 Pro）

## 适用场景
- 网络流、最短路径、最小生成树、最大匹配、社区发现
- 核心特征：问题是图/网络结构（节点+边+权重）

## 方法速查

| 问题类型 | 算法 | 复杂度 |
|---------|------|--------|
| 最短路径 | Dijkstra / Floyd | O(V²) |
| 最小生成树 | Kruskal / Prim | O(E log V) |
| 最大流 | Dinic / EK | O(V²E) |
| TSP/VRP | GA / SA / 蚁群 | NP-hard |
| 社区发现 | Louvain | O(N log N) |

## 完整Prompt模板（复制使用）

```
Context:
你正在解决一个数学建模竞赛的图论问题。题目涉及节点、边、权重、路径等网络结构。
你需要将实际问题转化为图论模型。

Objective:
建立图的模型→确定问题类型→选择算法→求解→可视化。

Constraints:
- 必须明确定义：节点代表什么、边代表什么、权重如何计算
- 如果问题NP-hard（如TSP），必须用启发式并比较至少2种方法
- 必须画网络结构图（节点+边+权重标注）
- 最优路径/流量/分布必须用图和表两种方式展示

Output Format:
1. 【图模型建立】
   节点定义：XX代表YY（数量N）
   边定义：XX代表YY（数量M）
   权重定义：由ZZ计算/题目给定
   
2. 【问题类型判断】
   属于XX问题，因为YY
   
3. 【算法选择】
   方法A — 理由 — 复杂度
   方法B — 理由 — 复杂度
   推荐：X
   
4. 【求解结果】
   最优值：XXX
   最优路径/方案：详细列出
   可视化：网络结构图（标注最优解）
```

## 常见坑
- 混淆有向图和无向图
- 权重定义不合理（如用距离却用时间的量纲）
- TSP直接用穷举（N>20就不行了）
- 网络图太丑或太大无法辨认

## 论文写作要点
- 必须画网络结构图（networkx+matplotlib）
- 路径结果用列表+图两种方式
- 算法复杂度分析写入正文
