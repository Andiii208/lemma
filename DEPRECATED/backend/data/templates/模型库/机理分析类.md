# 模型库 — 机理分析类（CO-STAR格式 · DeepSeek V4 Pro）

## 适用场景
- 物理过程、动力学系统、微分方程、仿真模拟
- 核心特征：有明确物理/生物/经济机理，可建立方程描述

## 方法速查

| 方法 | 工具 | 何时选用 |
|------|------|---------|
| ODE模型 | scipy solve_ivp | 连续时间过程 |
| PDE模型 | 有限差分法 | 空间+时间变化 |
| 状态空间 | 矩阵/线性代数 | 多变量控制系统 |
| 元胞自动机 | 网格+规则 | 空间扩散（火灾/交通/种群） |
| 蒙特卡洛 | 概率统计 | 不确定性强的随机过程 |

## 完整Prompt模板（复制使用）

```
Context:
你正在解决一个数学建模竞赛的机理分析问题。题目描述了一个物理/生物/经济过程，
有明确的机理可循。你需要建立数学模型描述这个过程。

Objective:
识别核心变量→建立数学关系（方程/ODE/PDE）→确定初始/边界条件→选择数值方法→求解→验证。

Constraints:
- 所有变量必须有物理定义和量纲
- 必须检查量纲一致性（dimensional analysis）
- 每个方程必须有物理意义解释
- 参数来源：文献值/数据拟合/物理常数
- 必须验证模型：解析解特例验证/数值解稳定性

Output Format:
1. 【核心变量识别】
   表格：变量 | 含义 | 量纲 | 来源
   
2. 【数学方程建立】
   方程1：(公式) — 物理意义
   方程2：(公式) — 物理意义
   ...
   
3. 【初始条件与边界条件】
   t=0: ...
   边界: ...
   
4. 【求解方法】
   解析解：{有/无}，{形式}
   数值方法：XXX（步长=XXX, 求解器=XXX）
   
5. 【模型验证】
   特例验证（参数取极值时的行为）
   与已知数据对比（如有）
   数值稳定性分析
   
6. 【参数敏感性】（🇸 论文灵魂）
   关键参数±20% → 输出变化曲线
```

## 常见坑
- 忘记量纲一致性检查
- 初始条件/边界条件定义不完整
- 数值方法不稳定（步长太大）
- 忽略关键相互作用项使模型过于简化
- 写"假设忽略XX"但没讨论忽略的合理性

## 论文写作要点
- 推导过程要详细，每步写出公式
- 引入无量纲化减少参数数量
- 解析解和数值解对比验证精度
- 参数来源要有依据
- 时间演化图是核心图表
