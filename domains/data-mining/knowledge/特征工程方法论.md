# 特征工程方法论

## 概述

特征工程是将原始数据转换为更适合建模的特征的过程。好的特征工程能显著提升模型性能，甚至比模型选择更重要。

## 核心概念

### 特征类型
- **数值特征**：连续值（温度、收入）和离散值（数量、次数）
- **类别特征**：名义型（城市、颜色）和有序型（学历、评分）
- **时间特征**：日期时间、时间戳、时间差
- **文本特征**：自由文本、类别描述
- **地理特征**：经纬度、行政区划

### 特征工程原则
- **业务优先**：特征应有业务含义解释
- **避免泄露**：预测时不可获得的信息不能作为特征
- **控制维度**：特征数不应远超样本数（除非使用正则化）
- **可复现**：特征构造逻辑必须记录并可复现

## 代码示例

### 类别特征编码

```python
import pandas as pd
from sklearn.preprocessing import LabelEncoder, OrdinalEncoder

def target_encode(train: pd.Series, target: pd.Series, test: pd.Series,
                  smoothing: float = 10) -> tuple:
    """Target Encoding，带平滑防止过拟合。"""
    global_mean = target.mean()
    stats = pd.DataFrame({'target': target, 'feature': train})
    agg = stats.groupby('feature')['target'].agg(['mean', 'count'])
    smooth = agg['count'] / (agg['count'] + smoothing)
    encoding = smooth * agg['mean'] + (1 - smooth) * global_mean
    return train.map(encoding), test.map(encoding).fillna(global_mean)
```

### 数值特征变换

```python
import numpy as np
from sklearn.preprocessing import PowerTransformer

def apply_transform(series: pd.Series, method: str = 'log') -> pd.Series:
    """对偏态分布进行变换。"""
    if method == 'log':
        return np.log1p(series)  # log(1+x) 避免 log(0)
    elif method == 'sqrt':
        return np.sqrt(series.clip(lower=0))
    elif method == 'boxcox':
        pt = PowerTransformer(method='box-cox')
        return pd.Series(pt.fit_transform(series.values.reshape(-1, 1)).flatten(),
                        index=series.index)
    return series
```

### 交互特征与多项式特征

```python
from sklearn.preprocessing import PolynomialFeatures

def create_interaction_features(df: pd.DataFrame, cols: list) -> pd.DataFrame:
    """为指定列创建交互特征。"""
    for i in range(len(cols)):
        for j in range(i + 1, len(cols)):
            col_name = f"{cols[i]}_x_{cols[j]}"
            df[col_name] = df[cols[i]] * df[cols[j]]
            ratio_name = f"{cols[i]}_div_{cols[j]}"
            df[ratio_name] = df[cols[i]] / df[cols[j]].replace(0, np.nan)
    return df
```

### 时间特征提取

```python
def extract_time_features(df: pd.DataFrame, col: str) -> pd.DataFrame:
    """从日期列提取丰富的时间特征。"""
    dt = pd.to_datetime(df[col])
    df[f'{col}_year'] = dt.dt.year
    df[f'{col}_month'] = dt.dt.month
    df[f'{col}_day'] = dt.dt.day
    df[f'{col}_weekday'] = dt.dt.weekday
    df[f'{col}_is_weekend'] = (dt.dt.weekday >= 5).astype(int)
    df[f'{col}_quarter'] = dt.dt.quarter
    df[f'{col}_day_of_year'] = dt.dt.dayofyear
    return df
```

### 特征选择

```python
from sklearn.feature_selection import mutual_info_classif, SelectKBest

def select_features_mi(X, y, k: int = 20) -> list:
    """基于互信息的特征选择。"""
    selector = SelectKBest(mutual_info_classif, k=k)
    selector.fit(X, y)
    mask = selector.get_support()
    return X.columns[mask].tolist()
```

## 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|----------|
| Target Encoding 泄露 | 用全局目标均值编码 | 在交叉验证 fold 内编码 |
| 高基数类别 | One-Hot 导致维度爆炸 | 使用 Target / Frequency Encoding |
| 忘记处理新类别 | 测试集出现训练集没有的类别 | 使用 `handle_unknown='ignore'` |
| 特征选择在 CV 外 | 选择过程引入信息泄露 | 将特征选择放入 Pipeline |
| 多重共线性 | 高度相关特征导致模型不稳定 | 计算 VIF，删除 >10 的特征 |

## 速查表

| 场景 | 推荐方法 | 关键参数 |
|------|----------|----------|
| 低基数类别 | OneHotEncoder | `handle_unknown='ignore'` |
| 有序类别 | OrdinalEncoder | `categories` 指定顺序 |
| 高基数类别 | TargetEncoding | `smoothing` 防过拟合 |
| 偏态数值 | log1p / Box-Cox | 检查变换后偏度 |
| 交互特征 | PolynomialFeatures | `degree=2, interaction_only=True` |
| 特征选择 | mutual_info / L1 | `k` 或 `C` 参数 |
| 高维降维 | PCA | `n_components` 按方差比选择 |
