# 摘要写作精要

## 结构化摘要四要素

摘要是论文的"电梯演讲"——在 200-300 字内让读者判断是否值得阅读全文。

### 1. 目的（Why）

说明研究的背景和动机，回答"为什么做这个研究"。

- 好的写法："针对传统方法在高维数据下计算复杂度高的问题..."
- 差的写法："本文研究了...问题"（空洞，无动机）

### 2. 方法（How）

概述采用的核心方法或技术路线，回答"怎么做的"。

- 好的写法："提出了一种基于改进 NSGA-II 的多目标优化算法，引入自适应变异算子和精英保留策略"
- 差的写法："采用了一些优化方法"（太模糊）

### 3. 结果（What）

呈现关键的量化结果，回答"得到了什么"。

- 好的写法："实验表明，所提算法在 ZDT1-ZDT4 测试集上的 IGD 指标平均降低了 23.5%，运行时间缩短了 40%"
- 差的写法："实验结果表明该方法具有良好的性能"（无数据支撑）

### 4. 结论（So What）

总结研究的意义或贡献，回答"这意味着什么"。

- 好的写法："该方法为大规模多目标优化问题提供了一种高效的求解框架"
- 差的写法："本文的研究具有重要意义"（空洞）

## 中英文摘要差异

| 方面 | 中文摘要 | 英文摘要 |
|------|---------|---------|
| 字数 | 200-300 字 | 150-250 词 |
| 时态 | 无时态变化 | 一般过去时（描述方法和结果），一般现在时（陈述结论） |
| 语态 | 主动语态为主 | 被动语态常见（"A method is proposed..."） |
| 第一人称 | "本文"、"我们" | "This paper"、"We" |
| 关键词 | 3-5 个，用分号分隔 | 3-5 个，用逗号分隔 |
| 格式 | 通常不分段 | 可分段（Background, Methods, Results, Conclusions） |

### 中英文对照示例

**中文摘要：**

> 针对城市交通信号配时优化问题，本文提出了一种基于深度强化学习的自适应控制方法。首先，构建了包含路口状态编码和动作空间定义的 Markov 决策过程模型；其次，设计了基于 Dueling DQN 的神经网络架构，引入优先经验回放机制加速训练。在 SUMO 仿真平台上的实验表明，与固定配时方案相比，所提方法使平均等待时间降低了 31.2%，通行量提升了 18.7%。结果证明深度强化学习在交通信号控制中具有显著优势。

**英文摘要：**

> This paper proposes an adaptive traffic signal control method based on deep reinforcement learning to address the urban traffic signal timing optimization problem. A Markov decision process model is first constructed with intersection state encoding and action space definition. A Dueling DQN-based neural network architecture is then designed with prioritized experience replay to accelerate training. Experiments on the SUMO simulation platform demonstrate that the proposed method reduces average waiting time by 31.2% and increases throughput by 18.7% compared with fixed-timing schemes. The results confirm the significant advantages of deep reinforcement learning in traffic signal control.

## 关键词选择策略

### 选择原则

1. **覆盖核心概念**：方法名、问题名、应用领域
2. **可检索性**：使用领域内通用术语，避免自造词
3. **具体性**：避免过于宽泛的词（如"研究"、"方法"）
4. **数量适当**：3-5 个为宜

### 关键词示例

| 论文主题 | 好的关键词 | 差的关键词 |
|---------|-----------|-----------|
| 交通信号优化 | 深度强化学习；交通信号控制；DQN；多目标优化 | 交通；优化；方法；研究 |
| 图像分类 | 卷积神经网络；迁移学习；图像分类；数据增强 | 图像；分类；深度学习 |
| 供应链优化 | 供应链管理；鲁棒优化；随机规划；库存控制 | 供应链；优化；管理 |

## 量化结果呈现技巧

### 具体数字优先

- 好："准确率达到 95.3%，较基线提升 4.2 个百分点"
- 差："准确率有显著提升"

### 对比基准明确

- 好："与 Random Forest 相比，F1-score 提升 5.8%"
- 差："优于现有方法"（哪个方法？提升多少？）

### 多维度指标

- 好："在准确率（96.1%）、召回率（94.7%）和推理时间（12ms/样本）上均优于对比方法"
- 差："综合性能更优"

### 效应量和统计显著性

- 好："配对 t 检验 p < 0.001，Cohen's d = 0.85，表明效果显著且效应量大"
- 差："效果显著"（统计依据是什么？）

## 常见错误

### 空洞表述

| 错误示例 | 改进方案 |
|---------|---------|
| "取得了良好的效果" | "准确率提升至 95.3%" |
| "具有重要的理论意义" | "为 XX 领域提供了新的建模框架" |
| "实验结果表明方法有效" | "在 5 个数据集上平均 F1 达到 0.92" |
| "本文的创新点如下" | 直接描述具体贡献 |

### 缺少数据

- 摘要中没有任何数字，全是定性描述
- 解决方案：每个关键结论至少附一个具体指标

### 过度承诺

- "彻底解决了 XX 问题" → 改为 "在 XX 场景下有效改善了 XX 指标"
- "首次提出" → 改为 "提出了一种新的..."（除非确实查证过）

### 方法描述过于详细

- 摘要不是正文，不需要列出所有步骤
- 只需点明核心创新，细节留给正文

### 摘要与正文不一致

- 摘要中提到的结果在正文中找不到支撑
- 摘要中的方法描述与正文实际内容不符

## 摘要写作模板

### 通用模板（中文）

```
[背景/问题] 针对 [具体问题] 中存在的 [具体困难/不足]，
[方法] 本文提出了一种基于 [核心技术] 的 [方法名称]。
[具体步骤1] 首先，[描述第一步]；
[具体步骤2] 其次，[描述第二步]。
[结果] 实验结果表明，[具体指标1] 达到 [数值]，
[对比] 较 [对比方法] 提升了 [百分比]；
[具体指标2] 为 [数值]。
[结论] 该方法为 [应用领域] 提供了 [具体贡献]。
```

### 通用模板（英文）

```
[Background] To address [specific problem] in [field],
[Method] this paper proposes [method name] based on [core technique].
[Step 1] First, [description].
[Step 2] Then, [description].
[Results] Experimental results show that [metric] reaches [value],
[Comparison] which is [percentage] higher than [baseline].
[Metric 2] achieves [value].
[Conclusion] The proposed method provides [contribution] for [application area].
```

### 数学建模论文摘要模板

```
本文针对 [赛题名称] 问题，建立了 [模型类型] 模型，提出了 [求解方法]。

针对问题一，[简述方法]，得到 [关键结果，含具体数值]。

针对问题二，[简述方法]，结果表明 [结论，含数据支撑]。

针对问题三，[简述方法]，[关键发现]。

最后，对模型进行了 [灵敏度分析/误差分析]，结果表明 [模型稳健性结论]。

关键词：[关键词1]；[关键词2]；[关键词3]；[关键词4]
```

## 检查清单

- [ ] 是否包含四要素（目的、方法、结果、结论）？
- [ ] 是否有至少 3 个具体数据/指标？
- [ ] 是否避免了空洞表述？
- [ ] 关键词是否 3-5 个、具体且可检索？
- [ ] 中英文摘要内容是否一致？
- [ ] 字数是否符合要求（中文 200-300 字，英文 150-250 词）？
- [ ] 是否与正文内容一致？
